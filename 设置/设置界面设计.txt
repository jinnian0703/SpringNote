偏好设置页面采用左右分栏布局设计。左侧为设置导航区域，以纵向列表形式展示所有配置分类，用户可快速切换不同设置页面。从上到下依次为：偏好设置、供应商、默认模型、快捷键、统计、关于。导航区域保持固定宽度，采用简洁的列表样式，通过选中状态高亮当前页面。

右侧为对应设置内容区域，根据左侧所选分类动态切换页面内容。其中“偏好设置”作为默认页面，整体采用纵向卡片式布局，并按照功能模块划分为三个独立设置区域，增强信息层级与可读性。

第一部分为个人信息模块，用于记录用户的基础工作信息。模块内包含每日工作时长、日薪以及所在行业三个配置项。每日工作时长用于统计用户的日常工作时间，并作为金币收益、工作统计等功能的计算依据；日薪用于计算工作收益与效率分析；所在行业用于优化 AI 总结内容与行业相关建议，使生成结果更符合用户的实际工作场景。

第二部分为字体设置模块，用于管理应用整体的显示字体与界面缩放。模块顶部为“应用字体”设置项，右侧显示当前选中的字体名称，默认状态为“系统默认”。用户点击字体选择框后，将弹出字体选择对话框。

字体选择弹窗用于修改整个应用界面的显示字体，包括页面标题、导航菜单、按钮文字、设置项文本以及普通内容区域文本。用户完成选择后立即生效，并自动保存配置，无需手动确认。

弹窗顶部显示标题“应用字体”，下方为字体搜索框，提示文字为“输入以过滤字体…”。搜索框支持字体名称模糊匹配与实时筛选，方便用户在大量字体中快速定位目标字体。

字体列表区域自动读取当前系统已安装的所有字体。每个字体项由字体名称与实时预览区域组成，预览区域显示“Aa字”示例文本，并使用对应字体实时渲染，使用户能够直观查看字体效果。列表支持滚动浏览与即时预览。

列表顶部固定提供“系统默认”选项。当用户选择该项时，应用将跟随当前操作系统默认字体，不再额外覆盖字体设置。Windows 系统默认使用 Segoe UI Variable，macOS 默认使用 SF Pro，Linux 默认使用 Noto Sans。

字体选择框右侧提供重置按钮。点击后将清除用户自定义字体配置，恢复系统默认字体设置，并立即刷新界面显示效果。

字体设置模块下方为“字体大小”设置项。右侧采用百分比输入框形式，默认值为 100%。用户可直接输入缩放比例调整界面文字大小，修改后实时生效。输入框右侧同样提供重置按钮，用于恢复默认的 100% 缩放比例。

第三部分为行为与启动模块，用于管理应用运行行为。模块包含“开机自启动”与“显示更新”两个开关选项。开机自启动用于控制应用是否随系统启动自动运行；显示更新用于控制应用是否在启动时自动检测并提示可用更新。所有设置均采用开关组件实现，并在用户操作后立即保存，无需额外确认步骤。

供应商页面用于统一管理应用接入的 AI 服务供应商、接口配置以及模型资源，整体采用三栏式布局设计，使用户能够快速完成供应商配置、模型管理以及接口测试等操作。

页面左侧为设置导航区域，采用纵向菜单布局，用于切换系统各项设置页面。从上到下依次包含偏好设置、供应商、默认模型、快捷键、统计以及关于等功能模块。其中当前页面“供应商”处于选中状态，并通过背景高亮进行视觉区分。

页面中间区域为供应商列表区域。顶部提供搜索框，支持根据供应商名称进行实时筛选。下方以列表形式展示所有已添加的供应商配置，每个供应商显示图标、名称以及当前状态。启用状态通过状态标签进行标识，方便用户快速查看当前可用的模型来源。

用户点击供应商后，右侧配置区域自动切换至对应供应商的详细配置页面。列表支持同时管理多个供应商，例如 OpenAI、Gemini、Claude、OpenRouter、本地模型服务以及其他兼容 OpenAI API 规范的第三方服务。

供应商列表底部提供“添加”按钮，用于创建新的供应商配置。

点击“添加”按钮后，页面中央弹出新增供应商对话框。对话框采用模态窗口设计，覆盖当前页面内容，引导用户完成供应商创建流程。

弹窗顶部显示标题“添加供应商”，右上角提供关闭按钮。标题下方提供供应商模板切换区域，用户可快速选择 OpenAI、Google、Claude 等常见服务商模板。选择模板后，系统自动填充对应的推荐配置，减少重复输入操作。

基础配置区域包含多个核心设置项。

“是否启用”用于控制当前供应商是否参与模型调用。当开关开启时，供应商创建完成后立即可用；关闭时仅保存配置，不出现在模型选择列表中。

“Use Responses API”用于控制是否使用新版 Responses API 接口。开启后自动使用对应接口规范；关闭时继续采用传统 Chat Completions 接口。

名称输入框用于设置供应商显示名称。该名称将在供应商列表、模型选择器以及相关设置页面中展示，支持用户自定义修改。

API Key 输入框用于填写服务商提供的访问密钥。输入内容默认采用密码隐藏显示，并支持后续修改。

Base URL 输入框用于配置接口服务地址。系统会根据当前供应商模板自动填充推荐地址，同时支持填写代理服务地址、自建服务地址以及本地模型服务地址。

API 路径输入框用于配置具体请求接口地址。默认采用兼容 OpenAI API 的聊天接口路径，同时支持自定义修改。

弹窗底部提供“添加”按钮。点击后系统自动校验配置内容，并创建新的供应商配置。创建成功后自动加入供应商列表，并切换至对应供应商详情页面。

页面右侧为供应商详细配置区域，用于管理当前选中供应商的接口信息与模型资源。顶部显示供应商名称，并提供启用开关。关闭后，该供应商下所有模型将不再参与模型调用。

基础配置区域包含 API Key、API Base URL 以及 API 路径三个核心配置项。所有修改均实时保存，无需额外点击保存按钮。

配置区域右上角提供“测试连接”功能。点击后系统自动验证当前 API 配置是否可正常访问，并反馈测试结果。连接成功时显示成功提示，连接失败时显示具体错误信息，帮助用户快速排查问题。

模型管理区域位于页面下方，用于维护当前供应商下的所有模型。顶部显示模型数量，并提供搜索模型、新增模型、删除模型、刷新列表以及自动获取模型等操作按钮。

点击“获取模型”后，系统自动调用当前供应商接口读取可用模型列表，并同步至本地配置。对于支持标准模型查询接口的服务商，无需手动录入模型名称即可完成模型导入。

模型列表采用纵向列表形式展示，每个模型显示模型名称与模型标识符。用户可快速查看当前供应商所提供的全部模型资源。

每个模型均提供独立操作功能，包括设为默认模型、编辑模型参数、高级配置以及删除模型。高级配置支持上下文长度、最大输出长度、温度参数、视觉能力、多模态能力以及推理能力等扩展设置。

所有供应商配置与模型配置均采用实时保存机制。用户完成修改后立即写入本地配置文件，无需额外保存操作，从而保证整个模型管理流程简单、直观且高效。

统计页面用于展示用户在应用中的整体使用情况、模型调用数据以及长期使用趋势，帮助用户了解自身工作习惯、AI 使用频率以及资源消耗情况。页面采用纵向信息布局设计，并通过时间筛选、热力图、统计卡片和趋势图四个层级逐步展示数据。

页面顶部为时间范围筛选区域，采用胶囊按钮形式展示不同统计周期。用户可在“全部”、“最近30天”、“上个月”、“上个季度”以及“自定义”之间快速切换。切换后，页面内所有统计数据、图表及趋势分析均同步更新，以反映对应时间范围内的实际使用情况。

当用户在“自定义时间段”弹窗中点击开始时间或结束时间输入框时，系统将弹出日期选择器，而非允许用户直接输入日期文本。

日期选择器采用月历视图展示当前月份日期，并提供月份切换功能。用户可通过顶部左右切换按钮快速浏览历史月份和未来月份，也可直接点击日期完成选择。

为了避免无效时间范围产生，日期选择器内置时间约束机制，实现选择过程中的实时校验，而非在用户点击“应用”后再进行错误提示。

当用户首先选择开始日期后，结束日期选择器将自动限制可选范围。所有早于开始日期的日期自动置灰并禁止点击，仅允许选择开始日期之后的日期作为结束时间。

同样，当用户优先选择结束日期时，开始日期选择器将自动限制可选范围。所有晚于结束日期的日期自动置灰并禁止选择，仅允许选择结束日期之前的日期作为开始时间。

不可选日期采用降低透明度的方式显示，并保持与正常日期明显区分。用户无法点击这些日期，从交互层面彻底避免“结束时间早于开始时间”等非法情况出现。

日期选择器支持跨月、跨季度以及跨年度选择。当时间跨度较大时，用户可通过顶部月份导航快速切换月份，无需逐日翻页。

完成日期选择后，对应日期自动回填至“开始时间”和“结束时间”输入区域，并即时更新当前选择状态。

由于日期选择阶段已经完成时间范围约束，因此点击“应用”按钮时无需再执行额外的合法性校验逻辑。系统可直接根据用户选择的时间区间刷新聊天热力图、统计总览以及用量趋势图等数据内容，从而提升整体交互效率并减少无效提示。

该方案采用“限制错误输入优于事后提示错误”的设计原则，通过禁用非法日期而非弹出报错信息，使整个时间筛选过程更加自然、流畅且符合现代桌面应用的交互习惯。


时间筛选区域下方为聊天热力图模块，用于展示用户长期使用活跃度。热力图参考首页的热力图 记录的是用户的commits次数

热力图下方为总览模块，用于展示核心统计指标。模块采用卡片式布局设计，每项指标以独立统计卡片形式展示，并突出关键数值。

总结数用于统计用户在首页使用智能生成的数量

编辑补全次数用于统计用户在便签页面触发编辑补全的次数

总记录数用于统计日报周报月报数量

日报数用于统计用户日报数量

周报数用于统计用户周报数量

月报数用于统计用户月报数量

输入 Tokens 用于统计用户向模型发送的全部 Token 数量，反映输入内容规模以及资源消耗情况。

输出 Tokens 用于统计模型生成内容所消耗的 Token 数量，用于衡量 AI 实际输出规模。

缓存 Tokens 用于统计命中缓存机制所节省或使用的 Token 数量，帮助用户了解缓存带来的成本优化效果。

应用启动次数用于统计应用被启动和打开的总次数，用于反映产品使用频率以及用户活跃程度。

所有统计卡片均支持根据当前筛选时间范围实时刷新，并自动更新对应数据。

页面底部为用量趋势模块，用于展示不同模型供应商或模型来源的使用变化情况。图表采用堆叠柱状图形式展示，每根柱体代表一个时间单位（按日、周或月统计），不同颜色对应不同供应商或模型来源。

用户可以直观看到 OpenAI、Gemini、Claude、本地模型以及其他供应商在不同时间段内的调用占比变化，了解自己最常使用的模型来源以及工作习惯变化趋势。

当鼠标悬停在柱状图上时，会弹出数据提示卡片，显示具体日期、供应商名称以及对应 Token 使用量，方便查看详细统计信息。
